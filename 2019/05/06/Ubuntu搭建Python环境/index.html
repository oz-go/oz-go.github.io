Ubuntu搭建Python环境

安装Python3、pip3

  1. Python3
    1. 安装命令
      1. sudo apt-get install python3
    2. 查找安装路径
      1. which python3
      2. /usr/bin/python3
  2. pip3
    1. 安装命令
      1. sudo apt-get install python3-pip
    2. 检查版本
      1. pip3 -V

安装mysql

  1. 安装命令
    1. sudo apt-get install mysql-server
    2. sudo apt-get install mysql-client
    3. sudo apt-get install libmysqlclient-dev
  2. 查看安装路径
    1. which mysql
    2. /usr/bin/mysql
  3. 查看mysql进程信息
    1. sudo netstat -tap | grep mysql
  4. 第一次使用mysql，设置密码
    1. 进入到etc/mysql 目录下，查看debian.cnf文件
    2. 找到用户名，密码 ，使用此账号登录mysql
    3. 修改root用户的的密码
      1. show databases；
      2. use mysql;
      3. update user set authentication_string=PASSWORD(“自定义密码”) where user=’root’;
      4. update user set plugin=”mysql_native_password”;
      5. flush privileges;
      6. quit;
    4. 修改完密码，需要重启mysql
      1. /etc/init.d/mysql restart;
    5. 再次登录
      1. mysql -u root -p 密码;

安装虚拟环境env

  1. 安装命令

    1. pip3 install virtualenv

    2. pip3 install virtualwrapper

    3. 编辑家目录下的.bashrc文件，添加下面两行

      1. export WIRKON_HOME=$HOME/.virtualenvs
      2. source ~/.local/bin/virtualenvwrapper.sh
      3. 以上是18.04版本ubuntu
      4. 16.04版本换成
        1. source /usr/local/bin/virtualenvwrapper.sh

    4. 创建虚拟环境

      1. mkvirtual -p python3 虚拟环境名

    5. 进入虚拟环境

      1. workon 虚拟环境名

    6. 退出虚拟环境

      1. deactivate

    7. 删除虚拟环境

      1. rmvirtualenv 虚拟环境名

    8. 安装问题

      1. 报错：/usr/bin/python: No module named virtualenvwrapper
        virtualenvwrapper.sh: There was a problem running the initialization hooks.If Python could not import the module virtualenvwrapper.hook_loader,
        check that virtualenvwrapper has been installed for
        VIRTUALENVWRAPPER_PYTHON=/usr/bin/python and that PATH is
        set properly.

      2. 这时候可以将VIRTUALENVWRAPPER_PYTHON路径(python全局路径)加入到~/.bashrc

        VIRTUALENVWRAPPER_PYTHON=/usr/bin/python

        再执行：source ~/.bashrc，成功！
